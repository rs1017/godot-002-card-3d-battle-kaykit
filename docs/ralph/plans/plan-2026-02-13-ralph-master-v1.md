## 1. 개요
### 1.0 작성 전략 (신풍 팁 반영)
1. 주제 폭 축소: 이번 문서는 `전투 가독성/즉시 피드백` 단일 주제에만 집중한다.
2. 플레이 경험 기반: Minion Masters 레퍼런스 관찰과 실제 전투 루프를 근거로 작성한다.
3. 신규+역기획 혼합: 신규 개선안 제안 + 기존 레퍼런스 동작 원리 역분석을 함께 사용한다.
4. 템플릿 가독성: 긴 문단 대신 표/플로우/요약 문장으로 핵심을 먼저 제시한다.

### 1.1 대상 게임 요약
| 항목 | 내용 |
|---|---|
| 장르 | 실시간 카드 배틀 (3D 보드형) |
| 핵심 루프 | 코스트 회복 -> 카드 사용 -> 전선 충돌 -> 결과 확인 |
| 플레이 목표 | 상대 핵심 오브젝트(타워/목표물) 파괴 |
| 현재 과제 | 전장 가독성, 즉시 피드백, 패배 원인 설명성 |

### 1.2 제안 배경 (Why)
현재 프로토타입은 "카드를 냈는데 무엇이 바뀌었는지" 체감이 약한 구간이 있고, 난전 시 정보 우선순위가 흔들린다.  
그래서 이번 기획은 기능 폭을 좁혀 다음 한 가지를 해결한다.

- 해결 과제: `카드 사용 직후 1~3초 내 전황 변화가 명확히 보이도록 피드백 계층과 HUD 우선순위를 재정의`

### 1.3 기대 효과
1. 초회 플레이어도 2분 내 핵심 규칙을 이해
2. 패배 후 "왜 졌는지"를 한 문장으로 설명 가능
3. 다수 유닛 교전 구간에서 시선 분산 감소

## 2. 본론
### 2.1 기획안 정의
이번 문서는 신규 시스템 전체가 아니라, 전투 중 `판독성 + 즉시 피드백`에 집중한 단일 개선안이다.

### 2.1-1 역기획 관찰 포인트
| 관찰 항목 | 레퍼런스 해석 | 본 프로젝트 적용 |
|---|---|---|
| 팀 구분 | 강한 색 대비로 소유권 인지 | 아군/적군 색상 체계 고정 |
| 전투 속도 | 소환 후 충돌까지 시간이 짧음 | 카드 사용 후 1~3초 결과 노출 |
| 정보 우선순위 | 코스트/핸드/핵심 체력이 먼저 보임 | HUD 우선순위 고정 |
| 난전 처리 | 이펙트가 많아도 주타겟 유지 | 중요 상태만 선별 노출 |

### 2.2 사용 시점 (언제 쓰는가)
1. 전투 초반: 코스트와 핸드 판단
2. 전투 중반: 다중 교전에서 타겟/상태 인지
3. 전투 후반: 승패 직전 의사결정

### 2.3 진행 플로우 (요약)
```text
[카드 선택]
   ->
[배치 확정]
   ->
[소환/진입]
   ->
[충돌/스킬]
   ->
[결과 피드백(피해/상태/오브젝트)]
   ->
[다음 의사결정]
```

### 2.5 상세 설계
#### A. HUD 우선순위
| 우선순위 | 요소 | 규칙 |
|---|---|---|
| 1 | 코스트 바 | 항상 시야 중앙 하단 기준 고정 |
| 2 | 핸드 | 코스트 부족/가능 상태를 색/명도로 즉시 구분 |
| 3 | 핵심 오브젝트 체력 | 화면 상단 고정, 피해 시 0.3초 강조 |
| 4 | 상태 이상/버프 | 유닛 상단 아이콘 최소화 + 중요 상태만 표시 |

#### B. 피드백 계층
| 계층 | 시간 | 예시 |
|---|---|---|
| 즉시 피드백 | 0~0.3초 | 히트 스파크, 타격 숫자, 짧은 사운드 |
| 상태 피드백 | 0.3~2초 | 둔화/방어감소 아이콘, 색상 링 |
| 결과 피드백 | 2초+ | 라인 밀림, 오브젝트 체력 변화, 승패 경고 |

#### C. 밸런싱 가이드
1. 저코스트 카드: 즉시 반응성 우선
2. 고코스트 카드: 전황 전환력 우선
3. 초반 스노우볼 방지: 초반 오브젝트 피해 보정 또는 진입 딜레이 최소 보정

### 2.6 보상/결과와 기대효과
| 구분 | 적용 전 | 적용 후 목표 |
|---|---|---|
| 카드 사용 체감 | 결과 지연 체감 | 1~3초 내 결과 인지 |
| 난전 구간 읽기 | 타겟 혼동 발생 | 아군/적군/핵심타겟 구분 유지 |
| 패배 해석 | 이유 모호 | 패배 원인 1문장 설명 가능 |

### 2.7 핵심 재미 요소 (한 문장)
카드를 내는 순간 전황이 바뀌는 손맛을 즉시 보여줘서, 짧은 판단-실행-보상 루프를 반복하게 만든다.

## 3. 구현 계획
### 3.1 작업 범위
- 씬: `scenes/battle/`, `scenes/ui/`
- 스크립트: `scripts/card_battle/`, `scripts/ui/`
- 데이터: `resources/cards/`

### 3.2 구현 단계
1. HUD 요소 우선순위 레이아웃 고정
2. 카드 사용 즉시 피드백 이벤트 훅 추가
3. 상태/결과 피드백 분리 적용
4. 수동 플레이로 가독성 점검 후 수치 조정

### 3.3 비범위 (이번 사이클 제외)
- 랭크/매치메이킹
- 다중 맵 운영
- 장기 성장/메타 시스템

## 4. QA/검증
### 4.1 필수 검증
1. `tools/qa_card_battle_static_checks.ps1` 통과
2. `tools/run_godot_safe.ps1 -Mode headless` 통과
3. 전투 씬 10분 수동 플레이 무크래시
4. 카드 사용 10회 테스트에서 결과 피드백 누락 0회

### 4.2 수용 기준
| 항목 | 기준 |
|---|---|
| 초기 이해도 | 첫 플레이 2분 내 규칙 파악 |
| 반응성 | 카드 사용 후 1~3초 내 전황 변화 인지 |
| 설명성 | 패배 직후 원인을 한 문장으로 기록 가능 |

### 4.3 리스크
1. 이펙트 강화가 오히려 가독성 저하를 유발할 수 있음
2. 모바일/저사양 대응 시 표시 밀도 축소 정책 필요
3. 카드별 예외 효과가 늘면 상태 아이콘 우선순위 규칙이 깨질 수 있음
